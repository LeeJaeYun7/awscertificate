IAM 정책이란?
- AWS 리소스에 대한 액세스 권한을 정의한 것
- 사용자, 그룹, 역할에 정책을 연결하여 사용
- JSON 문서 형식으로 이루어짐 

- JSON 문서에 명시되지 않은 정책은 거부 됨

- IAM 정책 JSON 문서 구조
(1) Effect - Allow, Deny
(2) Action
(3) Resource
(4) Condition

- EC2 서비스에 대한 정책

- 자격 증명 기반 정책
(1) 관리형 정책
- AWS 계정에 속한 다수의 사용자, 그룹 및 역할에 연결할 수 있는 정책
AWS 관리형 정책, 고객 관리형 정책
(2) 인라인 정책
- 단일 사용자, 그룹 또는 역할에 직접 추가하는 정책

- IAM 권한 경계
-> IAM 사용자 또는 역할에 최대 권한을 제한 하는 기능
-> AWS 전체 권한을 가지고 있어도 권한 경계에 대한 권한 범위로 축소되어 적용됨

- IAMFullAccess, IAMReadOnlyAccess
-> AWS 관리형 정책

- 사용자가 직접 정책을 생성할 수 있음

- 사용자에서 권한 추가를 통해 정책을 연결할 수 있음

- Mone.Kim에 AWSEC2FullAccess 권한을 부여
