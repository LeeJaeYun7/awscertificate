IAM 사용자
- 단일 개인 또는 애플리케이션에 대한 특정 권한을 가지고 있는 AWS 계정 내 자격 증명
- 한 사람과만 연관(한 명의 실제 사용자)
- 암호 또는 액세스 키와 같은 장기 자격 증명을 통해 액세스

- IAM은 글로벌 서비스
- 계정 설정을 누르면 암호를 설정할 수 있음
-> 암호 정책을 편집해서 보안을 강화할 수 있음

- 보안 자격 증명에서 MFA 할당 가능
-> MFA Devide로는 Authenticator app, Security Key, Hardward TOTP token 사용 가능

- 사용자를 만들어봄
-> 사용자 그룹 생성
-> 그룹을 만들고 그룹에 생성한 사용자를 할당함
-> Administrator Access 정책을 할당함
-> 사용자 생성을 함
-> 그룹에 가게 되면 IT Ops team이 만들어지고, Mone kim이 추가됨
-> Administrator Access 정책이 할당됨
-> JSON 형식

- 사용자를 클릭하면 권한이 그룹으로부터 상속됨
-> 콘솔 로그인

- 액세스 키 만들기를 클릭하고, 액세스 키를 여러 사용 사례들로 만들 수 있음
-> 로컬 코드를 클릭
-> 액세스 키가 만들어짐
-> 액세스 키와 비밀 액세스 키로 프로그래밍 방식으로 접속 가능함

- 사용자로 AWS 콘솔 로그인을 해봄
-> IAM 사용자로 로그인 가능
-> AdministratorAccess를 갖고 있어서, 어떤 페이지에 가서도 관리자 권한 수행 가능

- 그룹을 만들고, 그룹에 사용자를 만들어서 할당
-> IAM 사용자와 달리 역할은 누구나 맡을 수 있는 ID
