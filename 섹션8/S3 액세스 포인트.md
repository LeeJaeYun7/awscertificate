S3 액세스 포인트
-공유 버킷에 대한 액세스 문제점
-> 공유 버킷에 대한 액세스 관리
-> 여러 권한 수준을 사용하는 경우 수십 개에서 수백 개에 이르는 애플리케이션에 대한 액세스를 제어하는 단일 버킷 정책이 필요

- 애플리케이션이 더 커지면 버킷 정책은 더 복잡해지고, 관리하는 시간이 걸리며, 변경으로 인해 애플리케이션에 미치는 영향이 없는지
  확인하기 위해 감사가 필요

S3 액세스 포인트 사용
-고유한 액세스 포인트 정책을 가지는 네트워크 엔드포인트
-각 애플리케이션에 맞게 사용자 지정 이름과 권한으로 개별화된 액세스 포인트를 여러 개 사용
-허용된 S3 데이터만 액세스할 수 있게 가능

S3 다중 리전 액세스 포인트
- 여러 리전의 버킷 간의 자동 라우팅을 하는 액세스 포인트
- 액티브-액티브 S3 다중 리전 액세스 포인트
- 액티브-패시브 S3 다중 리전 액세스 포인트

- 단일 리전에서 사용하는 것과 같이 하나의 액세스 포인트를 사용
- AWS 글로벌 네트워크를 통해 라우팅 상태가 액티브인 가장 가까운 S3 버킷으로 자동으로 라우팅

- S3 액세스 포인트를 만들 수 있음

- 다중 리전 액세스 지점
