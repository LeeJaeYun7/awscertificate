S3의 버킷이나 객체에 대한 접근을 관리
- 3가지 유형이 있음

1) 버킷 정책
2) 퍼블릭 액세스 차단
3) ACL

1) 버킷 정책
- S3 버킷에 대한 액세스 제어 정책
- JSON 형식
- 조건에 따라 버킷에 액세스를 부여하고 제한 하는 기능으로 사용
ex) 특정 AWS 리소스에만 S3 버킷에 액세스 허용
    다른 AWS 계정에 버킷 액세스 권한을 부여 가능

2) 퍼블릭 액세스 차단
- 버킷과 객체에 대한 퍼블릭 액세스를 차단하거나 허용
- 데이터 보호를 위해 기본적으로 퍼블릭 액세스를 차단

3) ACL
- AWS 계정에 버킷이나 객체에 대한 읽기/쓰기 권한을 부여하는 기능
- 버킷 레벨에서 ACL을 적용하거나 객체 레벨에서 ACL을 적용 가능
- 각 객체에 대한 액세스를 개별적으로 제어하는 경우를 제외하고는 ACL
  비활성화 권장

- 권한
- 퍼블릭 액세스 차단
-> 정적 웹사이트 구축 시 
- ACL
- 읽기 쓰기 권한을 줄 수 있음
-> 이 기능을 사용하려면 IAM 권한 설정을 해야 함

- 버킷 정책을 통해서 S3 버킷에 대한 정책을 관리할 수 있음
-> 정책을 JSON 형식으로 만듦
