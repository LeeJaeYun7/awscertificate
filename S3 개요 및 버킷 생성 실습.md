데이터 저장 방식
(1) 오브젝트 스토리지
- 오브젝트라고 불리는 개별 유닛에 데이터를 저장하는 스토리지 포맷
- 각 유닛에는 고유의 식별자 혹은 키가 있어서 분산된 시스템 내 어디에 저장되어 있든지
  상관없이 데이터를 찾을 수 있음
- 각각의 오브젝트에는 키, 데이터 및 옵션 메타데이터가 포함
- Amazon S3

(2) 블록 스토리지
- 데이터를 고정된 사이즈의 블록으로 나누어 각각 고유한 식별자와 함께 저장하는 방식
- 각 데이터 블록은 고유 식별자를 부여 받아 스토리지 시스템이 데이터 조각을 원하는 곳에
  배치 가능
- Amazon EBS 
(3) 파일 스토리지
- 데이터를 계층적으로 저장
-> 디렉터리 내의 폴더에서 파일로 저장
-> 해당 데이터에 액세스해야 하는 경우, 컴퓨터는 그 데이터를 찾기 위해 경로를 알아야 함
-> Amazon EFS

S3 개요, 버킷 및 객체
- 거의 무제한 저장 용량을 제공하는 객체(Object) 스토리지 서비스
- 대부분의 경우 최소 3개의 가용영역에 데이터를 자동 분사 저장하기에 성능, 확장성, 가용성,
  내구성이 좋음
대표적 사용 사례
- 백업 및 복원, 재해 복구, 아카이브, 클라우드 스토리지, 정적 웹사이트 호스팅


버킷
- 오브젝트 저장공간
- 버킷은 리전 단위로 저장
- 버킷 이름은 유니크해야 함

객체
- 오브젝트(파일)
- 개별 오브젝트 최대 사이즈: 5TB

버킷+객체

버킷 만들기를 클릭해서 버킷을 생성
버킷에 객체를 업로드
해당 오브젝트를 클릭하면 오브젝트에 대한 상세한 내용이 나옴
방금 생성한 버킷에 대한 URL
오브젝트에 대한 이름
객체 URL을 복사해서 새 탭에 붙여넣음
-> 버킷이 Public Access가 차단됨

폴더 만들기
-> 폴더 안에 들어가서 객체를 추가로 업로드할 수 있음
